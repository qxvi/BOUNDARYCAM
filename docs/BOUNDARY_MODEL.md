# BOUNDARYCAM Boundary Model

BOUNDARYCAM starts from one operational question:

```text
What crossed the boundary?
```

A machine action is treated as boundary-relevant when it leaves an internal reasoning/output state and touches an external target, record, tool, workflow, registry, file, user, or decision surface.

BOUNDARYCAM separates the action into a Boundary Frame so the public viewer can see whether the action has authority, execution admission, evidence, replay, recognition, recourse, and closure.
