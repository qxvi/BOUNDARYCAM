# BOUNDARYCAM Operational Doctrine

Every machine action that touches the world needs a boundary frame.

A boundary frame records the boundary-facing facts of an action: actor, action, target, authority, execution, evidence, replay, recognition, recourse, and closure.

Allowed: public capture route, public frame model, public examples, public manifest, public schemas, public control state, public completion state.

Not allowed: private evidence claims, unverifiable proof claims, runtime enforcement claims without runtime wiring, hidden authority claims.
